# 1. 서비스 개요

## 1.1 프로젝트 배경

KBO 야구 팬 인구는 꾸준히 성장하고 있지만, 야구를 처음 접하는 입문자들은 "어떤 팀을 응원해야 하나?"라는 질문에서 막히는 경우가 많다. 기존에는 지역 연고나 주변 지인의 추천에 의존하는 것이 전부였다.

BallMate는 사용자의 성격·가치관·생활 스타일을 분석해 가장 잘 맞는 KBO 팀을 추천하고, 이어서 그 팀 안에서 나와 가장 잘 맞는 선수까지 추천하는 2단계 웹 서비스다.

## 1.2 서비스 한 줄 소개

***"성격과 취향으로 나에게 맞는 KBO 팀과 선수를 추천해주는 야구 입문 서비스"***

## 1.3 핵심 가치

- **입문자도 부담 없이 — 야구를 몰라도 생활 언어로 답할 수 있는 질문 설계**
- 근거 있는 추천 — 태그 기반 점수 매칭 알고리즘, 단순 랜덤이 아님
- 2단계 개인화 — 팀 추천에 이어 선수까지 매칭해 "내 선수"를 만들어줌
- 바이럴 설계 — 결과 공유 카드로 SNS 자연 확산 유도

# 2. 타깃 사용자

## 2.1 주요 타깃

| **구분** | **페르소나** | **특징** |
| --- | --- | --- |
| 1순위 | 야구 입문자 | 야구를 보고 싶은데 응원 팀·선수가 없는 상태 |
| 2순위 | 야구 데이트 희망자 | 야구장 데이트를 계획 중이나 어느 팀 경기를 볼지 모르는 커플 |
| 3순위 | 팀 재선택 고민자 | 기존 응원 팀이 있으나 나와 진짜 맞는지 확인하고 싶은 라이트 팬 |

## 2.2 사용자 여정

1. SNS 공유 카드 또는 지인 링크를 통해 서비스 진입
2. 랜딩 페이지에서 서비스 개요 확인 후 테스트 시작
3. 12문항 팀 추천 테스트 진행 (약 3분)
4. 결과 페이지에서 추천 팀 TOP 3 + 야구 페르소나 타입 확인
5. "이 팀 선수 추천받기" 버튼으로 선수 추천 플로우 진입
6. 7문항 선수 추천 테스트 진행 (약 2분)
7. 추천 선수 카드 확인 — 스탯응원가·스토리·직관 포인트
8. 결과 카드 SNS 공유 또는 다시하기

## 3.2 전체 플로우 (팀 → 선수 2단계)

팀 추천과 선수 추천은 독립적으로도 동작하지만, 자연스럽게 팀 결과 → 선수 추천으로 이어지도록 설계한다.

**랜딩  →  팀 테스트 (12문항)  →  팀 결과  →  선수 테스트 (7문항)  →  선수 결과 카드**

# 4. 팀 추천 질문 설계

## 4.1 설계 원칙

- 총 12문항, 약 3분 소요로 이탈 최소화
- Q1~Q5: 생활 밀착형 — 야구 지식 없이 답 가능한 질문
- Q6~Q9: 가치관성향 — 야구와 간접 연결
- Q10~Q11: 경기 스타일 — 처음 등장하는 야구 관련 질문
- Q12: 지역 선택 — 버튼형, 보조 가중치(×0.5)로 활용

## 4.2 전체 질문 목록

| **번호** | **카테고리** | **질문** | **왼쪽 (1점)** | **오른쪽 (5점)** | **척도** |
| --- | --- | --- | --- | --- | --- |
| Q01 | 생활 | 스트레스가 쌓이면 나는 보통 어떻게 푸나요? | 친구들이랑 소리 지르며 논다 | 혼자 조용히 생각을 정리한다 | 1-5 |
| Q02 | 생활 | 팀 프로젝트를 할 때 나는 어떤 스타일인가요? | 아이디어 내고 분위기를 이끈다 | 묵묵히 내 역할만 완벽하게 한다 | 1-5 |
| Q03 | 생활 | 여행을 간다면 어떤 여행이 더 끌리나요? | 핫플·맛집 활기찬 도심 | 조용한 자연·역사 있는 곳 | 1-5 |
| Q04 | 생활 | 응원하는 쪽이 지면 어떤 편인가요? | 감정이 상해 한동안 기억에 남는다 | 아쉽지만 금방 털어내는 편이다 | 1-5 |

| Q06 | 가치관 | 스포츠를 볼 때 무엇이 더 중요한가요? | 무조건 이기는 게 최고다 | 지더라도 서사와 과정이 중요하다 | 1-5 |
| Q07 | 가치관 | 어떤 팀을 오래 응원하고 싶은가요? | 매년 우승권에 있는 강팀 | 약하지만 성장하는 팀 | 1-5 |
| Q08 | 가치관 | 팀의 역사·전통이 얼마나 중요한가요? | 역사와 전통이 있어야 믿음직하다 | 역사보다 지금 잘하면 된다 | 1-5 |
| Q09 | 가치관 | 응원할 때 어떤 분위기를 좋아하나요? | 다 같이 소리 지르는 뜨거운 응원 | 조용히 경기에 집중하는 분위기 | 1-5 |
| Q10 | 경기 스타일 | 어떤 야구 경기가 더 재미있을 것 같나요? | 홈런 터지는 시원한 타격전 | 0점 싸움 쫄깃한 투수전 | 1-5 |

- 슬라이더 값 변환: 값 1 → 왼쪽 태그 ×2점 / 값 5 → 오른쪽 태그 ×2점 / 값 3 → 0점(중립)

# 9. 주의사항 및 리스크

## 9.1 저작권

- KBO 공식 로고, 선수 사진 무단 사용 금지 — 팀 컬러 CSS 카드로 대체
- 선수 사진: 구단 공식 제공 이미지 또는 직접 촬영본만 사용
- 응원가 가사: 구단별 저작권 상이 — 정식 서비스 전 각 구단 확인 필요
- 팀선수 소개 문구는 "서비스 내 추천 캐릭터" 톤으로 표현, 단정적 이미지 지양

## 9.2 데이터 사용

- KBO 공식 기록(타율, ERA 등) 크롤링 금지 — MVP에서는 하드코딩 텍스트만 사용
- 스탯 데이터는 시즌마다 갱신 필요 — 갱신 시 출처 명시
- 외부 데이터 연동 시 반드시 이용 조건 확인

# 10. 선수 추천 질문 설계 [v1.1 신규]

## 10.1 설계 원칙

팀 추천 12문항 이후 진행되는 2단계 테스트로, 피로감을 줄이기 위해 7문항으로 짧게 구성한다.

팀 추천이 생활 밀착형 질문(비야구)이었다면, 선수 추천은 포지션·플레이 스타일을 직접 묻는다. 단, 야구 모르는 사람도 이미지로 선택할 수 있도록 설명을 쉽게 풀어쓴다.

- 총 7문항, 약 2분 소요
- Q1 포지션 선택이 핵심 분기 — 타자 선택 시 Q2 활성화, 투수 선택 시 Q3 활성화 (실질 6문항)
- 객관식 버튼형 — 슬라이더 대신 명확하게 하나를 고르는 방식
- Q4~Q7은 포지션 무관 공통 질문

## 10.2 질문 목록

| **번호** | **카테고리** | **질문** | **선택지** |
| --- | --- | --- | --- |
| Q1 | 포지션 | 어떤 포지션 선수를 응원하고 싶나요? | 타자(공 치는 선수) / 선발투수(경기 시작하는 투수) / 마무리(끝 막는 투수) / 상관없다 |

| Q3 | 투수 스타일 | 투수라면 어떤 스타일이 더 멋있어 보이나요? (투수 선택 시만 노출) | 9이닝 혼자 막는 에이스 / 위기 때 나오는 마무리 / 빠른 공으로 삼진 / 변화구로 요리하는 스타일 |
| Q4 | 활약 상황 | 어떤 경기 상황에서 활약하는 선수가 더 끌리나요? | 결정적 순간에 터뜨리는 선수 / 매경기 꾸준히 잘하는 선수 |

| Q6 | 커리어 단계 | 선수의 나이와 커리어 단계 중 어디가 더 끌리나요? | 20대 초반 — 성장 지켜보는 재미 / 전성기 20대 후반~30대 초반 / 30대 중반 이상 — 베테랑의 연륜 |
| Q7 | 응원 기준 | 응원할 선수를 고를 때 가장 중요한 건 뭔가요? | 팀을 이끄는 존재감·카리스마 / 기록과 성적 / 경기장 밖 인간적인 모습 / 화려한 플레이 장면 |

# 11. 선수 데이터 및 추천 알고리즘 [v1.1 신규]

## 11.1 선수 태그 체계 (13개)

팀 태그와 별개로 선수 전용 태그 체계를 사용한다. 사용자 답변도 같은 태그 공간으로 변환되어 내적 점수로 매칭된다.

| **태그** | **분류** | **설명** |
| --- | --- | --- |
| power | 타격 | 홈런·장타 스타일, 파워 히터 |
| contact | 타격 | 안타·출루 스타일, 정교한 타격 |
| speed | 주루 | 빠른 발, 도루 능력 |
| defense | 수비 | 수비 중심 선수, 골든글러브급 |
| ace | 투수 | 팀 에이스, 선발 완투형 |
| closer | 투수 | 마무리·불펜, 위기 대응형 |
| clutch | 상황 | 득점권·결정적 순간에 강함 |
| consistent | 성적 | 꾸준한 성적, 변동 없는 퍼포먼스 |
| charisma | 존재감 | 팀 리더십, 카리스마, 존재감 |
| leader | 경험 | 베테랑, 오랜 경험, 팀 분위기 메이커 |
| young | 성장 | 유망주, 젊은 선수, 성장 서사 |
| underdog | 서사 | 고졸·저지명·자수성가 스토리 |
| foreign | 출신 | 외국인 선수 |

## 11.3 선수 추천 알고리즘

- Q1 포지션 선택 → 해당 포지션 태그 사전 가중치 부여 (타자: power·contact·speed·defense +1 / 투수: ace·closer +1)
- Q2~Q7 답변 → 선택지별 태그 점수 누적
- 팀 추천과 동일한 내적 방식으로 선수별 점수 계산
- 추천 팀 소속 선수 내에서만 순위 산정 → TOP 2 추천
- 매칭 퍼센트: 1위 선수 점수 대비 상대 비율

## 11.4 포지션 분기 로직

| **Q1 선택** | **스킵되는 문항** | **태그 사전 가중치** |
| --- | --- | --- |
| 타자 | Q3 (투수 스타일) | power·contact·speed·defense 각 +1 |
| 선발투수 | Q2 (타자 스타일) | ace·consistent +1 |
| 마무리 | Q2 (타자 스타일) | closer·clutch +1 |
| 상관없다 | Q2, Q3 모두 노출 | 사전 가중치 없음 |

# 12. 선수 결과 페이지 스펙 [v1.1 신규]

## 12.1 페이지 구성 요소

선수 결과 페이지는 7개 섹션으로 구성된다. 팀 컬러를 배경으로 사용해 몰입감을 높인다.

| **#** | **섹션** | **내용** |
| --- | --- | --- |
| 1 | 팀 배너 | 팀명 + 매칭 퍼센트 표시 |
| 2 | 선수 헤더 | 사진(플레이스홀더) + 이름 + 등번호 + 포지션 + 태그 뱃지 |
| 3 | 주요 스탯 | 타율·홈런·타점·OPS 또는 ERA·WHIP·탈삼진·이닝 (포지션별 다름) |
| 4 | 추천 이유 | "왜 나에게 맞나요" — 사용자 성향과 선수 태그 매칭 설명 + 관전 포인트 |
| 5 | 능력치 차트 | 6개 축 막대 바 (애니메이션) — 파워·클러치·타율·선구안·수비·스피드 등 |
| 6 | 응원가 | 선수 응원가 가사 + 멜로디 확인 안내 (저작권 주의) |
| 7 | 선수 스토리 | 출신·입단 배경·플레이 특징 요약 (2~3문장) |
| 8 | 홈구장 직관 포인트 | 구장명·위치·추천 좌석·관전 꿀팁 |
| 9 | 하단 버튼 | 결과 공유하기 / 다시하기 |

## 12.2 스탯 표시 기준 (포지션별)

| **포지션** | **표시 스탯 4개** |
| --- | --- |
| 타자·지명타자 | 타율(AVG) · 홈런(HR) · 타점(RBI) · 출루+장타율(OPS) |
| 선발 투수 | 평균자책점(ERA) · 이닝(IP) · 탈삼진(K) · WHIP |
| 마무리·불펜 | 평균자책점(ERA) · 세이브(SV) · 탈삼진(K) · 홀드(HLD) |

## 12.3 스탯 데이터 갱신 원칙

- MVP: 직전 시즌 스탯을 하드코딩 — KBO 공식 기록실 확인 후 입력
- v2 이후: 시즌 종료 시 수동 업데이트 또는 KBO 오픈 API 연동 검토
- 스탯 표기에 출처("KBO 공식 기록 기준") 명시 권장

## 12.4 선수 데이터 완성 우선순위

MVP에서는 추천 팀 상위 2~3팀의 선수 데이터만 먼저 완성한다. 전팀 완성은 v2 목표.

| **단계** | **대상 팀** | **기준** |
